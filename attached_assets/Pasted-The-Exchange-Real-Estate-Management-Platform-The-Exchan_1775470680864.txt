The Exchange — Real Estate Management Platform

The Exchange is a full-stack platform I built to simplify how independent estate agents manage properties, clients, and communication.

The idea came from a clear problem: smaller agencies often rely on a mix of spreadsheets, emails, and disconnected tools to manage tenants, documents, and rent tracking. This creates friction, poor visibility, and a fragmented experience for both agents and clients.

The Exchange brings all of this into a single system — giving agents a structured dashboard to manage their portfolio, while clients get a clean, guided portal to interact with their property, upload documents, and stay informed.

What it does

The platform is built around two connected experiences:

Agent Dashboard
Manage multiple properties and associated tenants
Onboard clients and invite them into the system
Define document requirements through structured checklists
Track rent schedules and payment status (paid / pending / overdue)
Create property-specific “Welcome Packs” with key information
Manage maintenance issues raised by tenants
Communicate directly with clients through built-in messaging
Client Portal
Complete onboarding through a guided document checklist
Upload and track required documents in real time
View rent schedules and payment details
Access property-specific information (Welcome Pack)
Report issues and communicate with their agent
Discover recommended local services (utilities, cleaning, etc.)
Why I built this

This project was designed to explore how fragmented real estate workflows can be unified into a single, intuitive system.

Rather than focusing on a narrow feature, I wanted to build something closer to a real product — combining:

data modelling (properties, clients, relationships)
document workflows
communication systems
role-based access (agent vs client)

The goal was to create something that could realistically be used by a small-to-medium agency.

Tech Stack
Frontend: React + TypeScript (Vite)
UI: Tailwind CSS, shadcn/ui, Radix UI, Framer Motion
Backend: Express.js (TypeScript)
Database: PostgreSQL (Drizzle ORM)
Auth: OpenID Connect (Replit Auth / Google OAuth)
Storage: Object storage with presigned uploads
State Management: TanStack React Query
Validation: Zod
Project Structure
client/   → React frontend  
server/   → Express backend + API  
shared/   → Database schema and shared types  
Running the project
npm install
npm run db:push
npm run dev

Environment variables are required for:

agent access control
agency branding
bank details
authentication
Notes
All sensitive configuration is handled via environment variables
Authentication is enforced across both portals
Data access is scoped (clients only see their own properties, agents only manage theirs)
File uploads are secured via presigned URLs
